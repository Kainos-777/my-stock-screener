# ALPHA DESK — 실시간 규칙 기반 종목 스크리너 + 백테스트 검증

장 마감 후(또는 언제든) 조회 시점 기준으로, **AI가 아닌 계산식**으로 단기 상승 가능성이 높아 보이는 종목을 선별해 보여주는 투자 참고용 도구입니다. 추가로, "화면에 뜨는 확률이 실제로 믿을 만한가?"를 과거 데이터로 직접 검증하는 **백테스트 기능**을 포함합니다.

## 이게 무엇인가

- Yahoo Finance에서 종목별 시세를 가져와 **RSI, 이동평균(5/20/60일), 거래량 비율, 단기 모멘텀, ATR 변동성**을 직접 계산합니다.
- 이 지표들을 가중 합산해 0~100점 점수를 매기고, 상위 N개를 정렬해 보여줍니다.
- "N영업일 내 상승 확률(추정)"은 점수를 40~82% 구간으로 환산한 **직관적 추정치**이며, 통계적으로 검증된 확률이 아닙니다.
- **백테스트 탭**에서 과거 1년 데이터로 이 추정치가 실제로 얼마나 맞았는지 직접 비교할 수 있습니다.
- Claude API나 다른 AI를 전혀 호출하지 않습니다. API 키도, 과금도 필요 없습니다.

## 백테스트 기능 — 어떻게 검증하는가

1. 과거 특정 날짜 T를 기준으로, **그 이전 데이터만 사용해** 그 시점의 점수를 계산합니다 (`lib/backtest.js`의 `scoreAsOf` — 미래 데이터를 절대 보지 않도록 슬라이스 처리, 미래 데이터를 조작해도 결과가 안 바뀜을 테스트로 직접 증명).
2. 점수가 "선정 기준" 이상이었던 경우만 모아서, 실제로 N영업일 후 주가가 올랐는지 비교합니다.
3. 인접한 채점 시점 간 결과 측정 구간이 겹치지 않도록(자기상관 방지) 간격을 둡니다.
4. "화면에 뜨는 예상 확률" vs "과거 실제 적중률"을 나란히 보여줘 괴리를 직접 확인할 수 있게 합니다.

## 한계 솔직히 공개 (5분야 전문가 교차검증 반영)

이 프로젝트는 풀스택 아키텍처, 보안, 개발, UI/UX, 주식(금융) 5개 분야 관점에서 두 차례(스크리너 본체, 백테스트 기능) 교차 검증을 거쳤고, 발견된 핵심 결함은 모두 수정했습니다. 주요 수정 이력:

**스크리너 본체:**
- **아키텍처**: market=ALL 스캔 시 서버리스 타임아웃 초과 가능 → 시간 예산 컷오프 + `partial_scan` 명시
- **금융 로직**: 목표가/손절가 손익비가 사실상 고정값이던 결함 → 신호 품질 기반 가변 손익비(1.3~2.8)로 재설계
- **보안**: rate limiting 전무 → IP당 제한 추가
- **코드 품질**: MACD가 부정확한 근사치였던 문제 → 정확한 골든/데드크로스 구현, 가중치를 `WEIGHTS` 설정 객체로 추출

**백테스트 기능 추가시:**
- **아키텍처**: 백테스트(더 무거운 작업)가 스크리닝보다 시간 예산을 더 길게 잡은 역설 → 6초로 보수화
- **개발**: 확률 환산 공식이 `screen.js`와 `backtest.js`에 중복 정의되어 한쪽만 고치면 검증 대상이 틀어지는 위험 → `lib/probability.js`로 단일화, 소스코드 레벨로 재발 방지 테스트 추가
- **UI/UX**: 모바일에서 핵심 비교 카드가 줄바꿈되어 메시지가 끊기던 문제, 백테스트 소요시간 예측 불가 → 반응형 레이아웃 + 종목 수 비례 예상시간 표시
- **보안**: 무거운 백테스트와 가벼운 스크리닝이 동일 rate limit을 공유해 실질 보호력이 약하던 문제 → 엔드포인트별 분리 한도(스크리닝 10회/분, 백테스트 4회/분)
- **주식(가장 심각)**: `stepDays`가 `horizonDays`보다 짧아 인접 표본의 결과 측정 구간이 최대 75% 겹치는 통계적 결함(표본 수 착시) → `stepDays = horizonDays`로 중첩 완전 제거 (표본이 정직하게 감소하지만 신뢰할 수 있는 값이 됨)

여전히 남아있는 한계 (사용 전 인지 필요):

1. **종목 유니버스가 정적 리스트입니다** (data/universe.js, 약 140개 — 한국 시총 상위 + 미국 대형주).
   실시간 전수 스캔(코스피+코스닥 약 2,500개)은 무료 호스팅의 함수 실행시간 제한(보통 10~30초) 때문에 불가능합니다.
   분기 1회 정도 수동으로 리스트를 갱신하거나, 추후 거래소 랭킹 API로 자동화하는 것을 권장합니다.

2. **Yahoo Finance는 비공식 API입니다.** 공식 지원이 종료된 지 오래됐고, 가끔 차단되거나 비정상 응답(예: "Edge: Not Found")을 줍니다.
   lib/yahoo.js가 이런 상황을 방어적으로 처리해 해당 종목만 건너뛰지만, 완전한 가동률을 보장하지는 않습니다.

3. **백테스트에는 생존편향(survivorship bias)이 있습니다.** 백테스트 대상이 "현재" 시총 상위 종목 리스트이므로, 1년 전 기준으로는 순위가 달랐거나 그 사이 크게 하락·상장폐지된 종목은 애초에 검증 대상에서 빠집니다. 즉 결과가 실제보다 다소 낙관적으로 나올 수 있습니다 (화면에도 경고 문구로 고지됩니다).

4. **백테스트는 신호 품질(`riskRewardRatio` 등)을 함께 검증하지는 않습니다.** "선정 여부"와 "N일 후 상승 여부"만 비교하며, 손익비·변동성 조정 후 실질 수익률까지는 다루지 않습니다.

## 프로젝트 구조

api/screen.js        서버리스 함수 — GET /api/screen?market=KR_ALL&limit=15&sector=반도체
api/backtest.js       서버리스 함수 — GET /api/backtest?market=KOSPI&count=12&minScore=60&horizonDays=10
data/universe.js      종목 유니버스 (정적 시드 리스트)
lib/indicators.js     RSI/SMA/EMA/MACD/ATR 등 순수 계산 함수
lib/scoring.js        지표를 조합해 0~100점 산출 (WEIGHTS 설정 객체로 가중치 분리)
lib/probability.js     점수→확률 환산 공식 (screen.js와 backtest.js가 공유하는 단일 진실 공급원)
lib/backtest.js        백테스트 엔진 (lookahead bias 방지 설계)
lib/yahoo.js           Yahoo Finance 호출 + 방어적 파싱
lib/rateLimit.js       엔드포인트별 분리 rate limiting
public/index.html      프론트엔드 (스크리닝 탭 + 백테스트 탭)
test/                  전체 단위/통합 테스트 (150개 이상)

## 로컬에서 테스트 실행

npm install
npm test

11개 테스트 스위트(지표 계산, MACD, 백테스트 엔진, Yahoo 파서, 파이프라인, 시간예산, rate limit, 스크리닝 API, 백테스트 API, 프론트엔드 렌더링×2)가 전부 통과해야 정상입니다 (총 150개 이상 개별 검증).

## GitHub + Vercel로 배포하기

1. 이 폴더를 GitHub repo로 push

   git init
   git add .
   git commit -m "init: rule-based stock screener"
   git branch -M main
   git remote add origin https://github.com/계정명/repo이름.git
   git push -u origin main

2. vercel.com 가입 후 "Add New Project" → 방금 만든 GitHub repo 선택 → Import

3. 별도 환경변수 설정 없이 바로 Deploy 가능합니다 (API 키 불필요).

4. 배포 완료 후 https://프로젝트명.vercel.app 으로 접속하면 바로 작동합니다.

## (선택) 나중에 AI 설명 기능을 추가하고 싶다면

api/explain.js를 새로 만들어 상위 15개의 계산된 지표값을 Claude API에 보내 자연어 설명만 받아오는 방식을 권장합니다.
이 경우 Vercel 프로젝트 설정의 Environment Variables에 ANTHROPIC_API_KEY를 추가하고, 프론트엔드가 아닌 서버리스 함수 안에서만 키를 사용해야 합니다 (브라우저에 노출 금지).
